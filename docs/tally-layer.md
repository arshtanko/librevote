# Tally Layer

The MVP tally layer recomputes simple deterministic results from locally valid objects.

## Trustee Selection Tally

Inputs:

- Valid `TrusteeNomination` objects.
- Valid non-conflicted `TrusteeVote` objects.

Output:

- `TrusteeSelectionResult` with deterministic `candidate_ranking[]`.

Tie-breaking must be deterministic, for example by candidate id or object id.

## Final Trustee Set

The final MVP trustee set is derived from:

- `candidate_ranking[]` from `TrusteeSelectionResult`.
- Valid non-conflicted `TrusteeConsent` objects.
- The configured trustee count required by the current schema or service.

## Key Set

`TallyKeySet` is generated by deterministic placeholder logic from the final trustee set and valid `TallyKeyContribution` objects. It marks the ballot phase as ready.

## Ballot Tally

Inputs:

- Valid non-conflicted `AnonymousBallot` objects.
- Active `AnonymousElection` with accepted `TallyKeySet`.

Output:

- `TallyResult` with per-choice counts.

Invalid or conflicted ballots are excluded. Any invalid ballot count is local diagnostic information.
